# AFMC Mess App (Basic Starter)

Basic starter structure for AFMC Mess Management Web Application.

## Stack
- React.js (client)
- Express.js (server)
- MySQL (database)

## Project Structure

### Root Directory
- `client/`: Frontend React application
- `server/`: Backend Express.js server
- `database/`: Database schema and sample data

### Client Folder Architecture
The `client/` directory contains the React frontend application:

- `build/`: Production build files (generated by build process)
  - `asset-manifest.json`: Asset manifest for the build
  - `index.html`: Main HTML file for the built app
  - `static/`: Static assets (CSS, JS)
    - `css/`: Compiled CSS files
    - `js/`: Compiled JavaScript files
- `public/`: Static public assets served directly
  - `index.html`: Template HTML file
- `scripts/`: Custom build and utility scripts
  - `fix-html5-qrcode-sourcemaps.js`: Script to fix QR code sourcemaps
  - `start-react-scripts.js`: Custom start script for React
- `src/`: Source code for the React application
  - `App.js`: Main App component
  - `index.css`: Global CSS styles
  - `index.js`: Entry point for the React app
  - `assets/`: Static assets (images, icons, etc.)
  - `components/`: Reusable React components
    - `Loader.jsx`: Loading spinner component
    - `Navbar.jsx`: Navigation bar component
    - `OrderDetailsModal.jsx`: Modal for order details
    - `ProtectedRoute.jsx`: Route protection component
    - `Sidebar.jsx`: Sidebar navigation component
    - `Table.jsx`: Table component for displaying data
    - `admin/`: Components specific to admin functionality
    - `attendant/`: Components for attendant role
    - `common/`: Shared/common components
    - `kitchen/`: Components for kitchen operations
    - `layouts/`: Layout components (e.g., page layouts)
    - `user/`: Components for end-user interface
  - `context/`: React context providers for state management
  - `pages/`: Page-level components (full page views)
  - `routes/`: Routing configuration and route definitions
  - `services/`: API service functions for backend communication
  - `utils/`: Utility functions and helpers

### Database Folder Architecture
The `database/` directory contains database-related files:

- `sample_data.sql`: Sample data for populating the database
- `schema.sql`: Database schema definition
- `seed.sql`: Seed data for initial setup

## Concepts and Technologies

This AFMC Mess Management Application implements a comprehensive full-stack solution for managing mess operations. Below are the key concepts and technologies used:

### Frontend Technologies (React.js)

#### Core Framework
- **React 18**: Modern React with hooks, functional components, and concurrent features
- **JSX**: JavaScript XML for component templating
- **React Router DOM**: Client-side routing for single-page application navigation

#### State Management
- **React Context API**: Global state management for authentication and user data
- **React Hooks**: useState, useEffect, useContext for component state and lifecycle

#### Form Handling & Validation
- **React Hook Form**: Efficient form state management and validation
- **Zod**: TypeScript-first schema validation for form data
- **@hookform/resolvers**: Integration between React Hook Form and Zod

#### UI/UX Libraries
- **Tailwind CSS**: Utility-first CSS framework for responsive design
- **PostCSS**: CSS processing and optimization
- **Lucide React**: Modern icon library
- **React Icons**: Additional icon collections
- **SweetAlert2**: Beautiful modal dialogs and alerts
- **React Toastify**: Toast notifications for user feedback

#### HTTP Communication
- **Axios**: Promise-based HTTP client for API requests
- **JWT Decode**: Client-side JWT token parsing and validation

#### Specialized Features
- **HTML5 QR Code**: QR code scanning functionality
- **jsPDF & jsPDF-AutoTable**: PDF generation and table formatting
- **html2canvas**: HTML element to canvas conversion for screenshots
- **Day.js**: Lightweight date manipulation library

#### Development Tools
- **Create React App**: Build setup and development server
- **React Testing Library**: Component testing utilities
- **ESLint**: Code linting and style enforcement

### Backend Technologies (Express.js)

#### Core Framework
- **Express.js 5**: Fast, unopinionated web framework for Node.js
- **Node.js**: JavaScript runtime for server-side development

#### Database
- **MySQL2**: MySQL database driver with promise support
- **Relational Database Design**: Structured data storage with foreign keys and relationships

#### Authentication & Security
- **JSON Web Tokens (JWT)**: Stateless authentication mechanism
- **bcryptjs**: Password hashing for secure storage
- **Helmet**: Security middleware for HTTP headers
- **CORS**: Cross-origin resource sharing configuration

#### API Design
- **RESTful API**: Representational State Transfer architecture
- **Express Router**: Modular route organization
- **Middleware Pattern**: Request/response processing pipeline

#### Data Validation
- **Express Validator**: Server-side input validation and sanitization

#### File Handling
- **Multer**: Middleware for handling multipart/form-data (file uploads)
- **Static File Serving**: Express static middleware for uploads directory

#### Communication
- **Nodemailer**: Email sending functionality for notifications

#### Logging & Monitoring
- **Morgan**: HTTP request logger middleware
- **Winston**: Advanced logging framework with multiple transports

#### Development Tools
- **Nodemon**: Automatic server restart during development
- **Dotenv**: Environment variable management

### Database Concepts

#### Schema Design
- **Users Table**: Authentication and role-based access control
- **Orders System**: Order processing, items, and transactions
- **Inventory Management**: Stock tracking and management
- **Menu Management**: Food/drink items and pricing
- **Payment Processing**: Transaction handling
- **Reporting**: Analytics and profit management
- **Notifications**: System messaging
- **Offers/Promotions**: Discount and special pricing

#### Data Relationships
- **Foreign Keys**: Referential integrity between tables
- **Joins**: Complex queries combining multiple tables
- **Indexes**: Performance optimization for frequently queried columns

### Application Architecture Patterns

#### MVC Pattern (Backend)
- **Models**: Database interaction and data modeling
- **Views**: JSON responses (API endpoints)
- **Controllers**: Business logic and request handling

#### Component-Based Architecture (Frontend)
- **Reusable Components**: Modular UI building blocks
- **Container/Presentational Pattern**: Separation of logic and presentation
- **Higher-Order Components**: Code reuse through component wrapping

#### Role-Based Access Control (RBAC)
- **User Roles**: Admin, Attendant, Kitchen, User (End-user)
- **Route Protection**: Middleware-based access control
- **Permission Levels**: Different functionality based on user role

#### Service Layer Pattern
- **Service Classes**: Business logic separation
- **Helper Functions**: Utility operations
- **Middleware**: Cross-cutting concerns (auth, error handling, logging)

### Business Logic Concepts

#### Mess Management Features
- **Authentication System**: Login/logout with role redirection
- **Inventory Tracking**: Stock levels, alerts, and reporting
- **Menu Management**: Dynamic menu creation and pricing
- **Order Processing**: Order creation, modification, and cancellation
- **Payment Integration**: Transaction processing and records
- **Reporting & Analytics**: Sales reports, profit analysis, stock reports
- **Notification System**: Alerts for low stock, order status, etc.
- **Member Management**: User profiles and preferences
- **Offer Management**: Promotions and discounts
- **Cocktail Management**: Specialized drink ordering system

#### Workflow Concepts
- **Order Lifecycle**: Creation → Processing → Completion/Cancellation
- **Inventory Flow**: Stock in → Consumption → Reordering alerts
- **Payment Flow**: Order → Payment → Confirmation
- **Reporting Cycle**: Data collection → Analysis → Insights

### Security Concepts

#### Authentication
- **Password Hashing**: Secure password storage with bcrypt
- **JWT Tokens**: Stateless session management
- **Token Expiration**: Automatic logout after inactivity

#### Authorization
- **Role-Based Permissions**: Different access levels per user type
- **Route Guards**: Protected endpoints with middleware
- **Input Validation**: Both client and server-side validation

#### Data Protection
- **SQL Injection Prevention**: Parameterized queries
- **XSS Protection**: Input sanitization
- **CORS Configuration**: Controlled cross-origin access

### Development Practices

#### Code Organization
- **Modular Structure**: Separated concerns (routes, controllers, models)
- **Utility Functions**: Reusable helper code
- **Configuration Management**: Environment-based settings

#### Error Handling
- **Try-Catch Blocks**: Exception management
- **Error Middleware**: Centralized error processing
- **HTTP Status Codes**: Appropriate response codes

#### Testing
- **Unit Testing**: Component and function testing
- **Integration Testing**: API endpoint testing
